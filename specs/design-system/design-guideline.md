## 設計理念與原則

Town Pass 設計系統建立在**簡潔、直觀、一致**的核心理念之上，旨在為使用者提供流暢且愉悅的數位體驗。

### 核心價值
- **一致性**：確保所有接觸點的體驗統一
- **可及性**：讓所有使用者都能輕鬆使用
- **效率性**：幫助使用者快速完成任務
- **美觀性**：創造視覺愉悅的使用體驗

## 色彩系統 Color System

### 主要色彩 Primary Colors

#### Primary 主色系（青藍色）
主色系代表品牌的核心識別，用於主要操作按鈕、連結和重要的互動元素。

| 色階 | 色碼 | 使用情境 |
|------|------|----------|
| Primary/50 | #EDF8FA | 背景色、hover 狀態的淺色背景 |
| Primary/100 | #DBF1F5 | 輔助背景、標籤背景 |
| Primary/200 | #B4E2EA | 邊框、分隔線 |
| **Primary/500** | **#5AB4C5** | **主要按鈕、連結、品牌色** |
| Primary/600 | #468D9B | Hover 狀態、強調色 |
| Primary/700 | #356C77 | Active 狀態、深色文字 |
| Primary/900 | #112629 | 標題文字（深色模式） |

#### Secondary 次要色系（金黃色）
次要色系用於輔助性的操作和視覺重點，創造視覺層次。

| 色階 | 色碼 | 使用情境 |
|------|------|----------|
| Secondary/50 | #FDF8ED | 提示背景、高亮區域 |
| Secondary/200 | #F8E3BC | 警示背景（輕度） |
| **Secondary/500** | **#F5BA4B** | **次要按鈕、圖標、強調元素** |
| Secondary/700 | #AD7B2B | Hover 狀態 |

### 語意色彩 Semantic Colors

語意色彩用於傳達特定的狀態和訊息：

#### ✅ Success 成功
- **主色**：#76A732
- **使用情境**：成功提示、完成狀態、正向回饋
- **搭配**：可搭配淺綠背景 rgba(118, 167, 50, 0.1)

#### ⚠️ Warning 警告
- **主色**：#FD853A (Orange/500)
- **輔色**：#F4B992 (Orange/300) - hover 狀態
- **使用情境**：警告訊息、需要注意的內容、中等優先級提醒

#### ❌ Danger 危險
- **主色**：#D45251 (Red/500)
- **輔色**：#E29494 (Red/300) - hover 狀態
- **使用情境**：錯誤訊息、刪除操作、高優先級警告

#### 🚫 Disabled 禁用
- **主色**：#E3E7E9 (Grey/100)
- **使用情境**：禁用的按鈕、不可用的選項、唯讀狀態

### 文字色彩 Text Colors

| 類型 | 色碼 | 使用情境 | 對比度要求 |
|------|------|----------|------------|
| 主要文字 | #30383D | 標題、正文主要內容 | 7:1 (AAA) |
| 次要文字 | #475259 | 副標題、輔助說明 | 4.5:1 (AA) |
| 說明文字 | #91A0A8 | 提示文字、時間戳記 | 3:1 (AA Large) |

### 背景色彩 Background Colors

| 類型 | 色碼 | 使用情境 |
|------|------|----------|
| 主背景（淺色） | #FFFFFF | 內容區域、卡片背景 |
| 次背景（灰色） | #F1F3F4 | 頁面背景、區塊分隔 |

### 色彩使用準則

#### ✅ Do's（正確做法）
- 使用主色系引導使用者注意重要操作
- 保持色彩使用的一致性
- 確保文字與背景的對比度符合 WCAG 2.1 標準
- 使用語意色彩傳達狀態信息

#### ❌ Don'ts（錯誤做法）
- 避免在同一頁面使用過多顏色（建議不超過 5 種）
- 不要僅依賴顏色傳達重要信息
- 避免使用高飽和度顏色作為大面積背景
- 不要忽視色盲使用者的需求

## 字體系統 Typography

### 字體家族
**主要字體**：PingFang SC（蘋方-簡）
**備用字體**：-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif

### 字體層級

| 層級 | 字重 | 字號 | 行高 | 使用情境 |
|------|------|------|------|----------|
| **H1 Semibold** | 600 | 36px | 48px | 頁面主標題、Hero 區域標題 |
| **H2 Semibold** | 600 | 24px | 32px | 區塊標題、對話框標題 |
| **Body Semibold** | 600 | 14px | 20px | 按鈕文字、標籤、強調內容 |
| **Body Regular** | 400 | 14px | 20px | 正文內容、表單輸入、一般文字 |

### 擴展字體規格（建議）

| 層級 | 字重 | 字號 | 行高 | 使用情境 |
|------|------|------|------|----------|
| Display | 700 | 48px | 64px | 行銷頁面、Banner |
| H3 | 600 | 20px | 28px | 小節標題、卡片標題 |
| H4 | 600 | 16px | 24px | 列表項目標題 |
| Caption | 400 | 12px | 16px | 輔助說明、時間戳記 |
| Small | 400 | 11px | 14px | 版權聲明、註腳 |

### 文字使用準則

#### ✅ Do's
- 保持文字層級的一致性
- 確保適當的行距以提升可讀性（1.4-1.6 倍）
- 正文寬度控制在 45-75 個字符之間
- 使用適當的字重創造視覺層次

#### ❌ Don'ts
- 避免在同一頁面使用超過 3 種字號
- 不要使用小於 11px 的字體
- 避免使用過長的文字行（超過 80 個字符）
- 不要在淺色背景上使用淺色文字

## 間距系統 Spacing System

### 基礎間距單位
Town Pass 採用 4px 為基礎單位的間距系統，確保視覺的一致性和韻律感。

| 名稱 | 數值 | 使用情境 |
|------|------|----------|
| **space-xs** | 4px | 圖標與文字間距、緊密排列 |
| **space-sm** | 8px | 表單元素內間距、小型元件 |
| **space-md** | 16px | 預設間距、卡片內間距 |
| **space-lg** | 24px | 區塊間距、大型元件內間距 |
| **space-xl** | 32px | 主要區塊分隔、頁面邊距 |
| **space-2xl** | 48px | 大型區塊間距、Section 分隔 |
| **space-3xl** | 64px | 頁面級別間距、Hero 區域 |

### 間距應用規則

#### 元件內間距（Padding）
- **按鈕**：垂直 8px，水平 24px
- **卡片**：24px
- **輸入框**：垂直 8px，水平 16px
- **對話框**：32px

#### 元件間間距（Margin）
- **同組元件**：8px
- **不同組元件**：16px
- **區塊間**：32px
- **頁面section**：48px

### 響應式間距

| 裝置 | 基礎間距 | 頁面邊距 | 網格間距 |
|------|----------|----------|----------|
| Mobile | 4px | 16px | 16px |
| Tablet | 4px | 24px | 24px |
| Desktop | 4px | 32px | 24px |
| Large Desktop | 4px | 48px | 32px |

## 元件規範 Component Guidelines

### 按鈕 Buttons

#### 按鈕類型
1. **Primary Button 主要按鈕**
   - 背景：Primary/500 (#5AB4C5)
   - 文字：白色
   - Hover：Primary/600
   - 用途：主要操作、CTA

2. **Secondary Button 次要按鈕**
   - 背景：Secondary/500 (#F5BA4B)
   - 文字：白色
   - Hover：Secondary/600
   - 用途：次要操作

3. **Outline Button 外框按鈕**
   - 背景：透明
   - 邊框：Primary/500 (2px)
   - 文字：Primary/500
   - Hover：背景 Primary/50
   - 用途：第三層級操作

#### 按鈕尺寸
| 尺寸 | 高度 | 內距 | 字體 |
|------|------|------|------|
| Large | 48px | 12px 32px | 16px |
| Medium | 40px | 8px 24px | 14px |
| Small | 32px | 6px 16px | 12px |

#### 按鈕狀態
- **Default**：預設狀態
- **Hover**：滑鼠懸停，顏色加深，上移 1px
- **Active**：點擊狀態，顏色更深
- **Disabled**：禁用狀態，背景 Grey/100，文字 Grey/400
- **Loading**：載入中，顯示旋轉動畫

### 表單 Forms

#### 輸入框 Input
- 高度：40px
- 邊框：1px solid Grey/200
- 圓角：8px
- 內距：8px 16px
- Focus：邊框 Primary/500 + 陰影

#### 標籤 Label
- 字體：Body Semibold
- 顏色：Text Primary
- 間距：底部 8px

#### 錯誤提示
- 文字顏色：Red/500
- 字體大小：12px
- 位置：輸入框下方 4px

### 卡片 Cards

#### 基礎卡片
- 背景：白色
- 邊框：1px solid Grey/100
- 圓角：12px
- 內距：24px
- 陰影：0 4px 6px rgba(0,0,0,0.05)

#### 可互動卡片
- Hover：陰影加深，上移 2px
- 過渡：all 0.2s ease

### 提示 Alerts

#### 結構
- 內距：16px 24px
- 圓角：8px
- 左邊框：4px solid
- 圖標：左側 16px

#### 類型
1. **Success**：背景 Green/50，邊框 Green/500
2. **Warning**：背景 Orange/50，邊框 Orange/500
3. **Error**：背景 Red/50，邊框 Red/500
4. **Info**：背景 Primary/50，邊框 Primary/500

## 設計原則 Design Principles

### 1. 清晰 Clarity
- 資訊架構清晰，讓使用者快速理解
- 使用明確的視覺層次引導注意力
- 文字簡潔明瞭，避免專業術語

### 2. 一致 Consistency
- 相同功能使用相同的設計模式
- 保持視覺風格的統一性
- 交互行為符合使用者預期

### 3. 效率 Efficiency
- 減少使用者完成任務的步驟
- 提供智能預設和自動完成
- 優化載入時間和響應速度

### 4. 美觀 Aesthetics
- 保持視覺的簡潔優雅
- 適當的留白創造呼吸感
- 精心設計的動畫提升體驗

### 5. 包容 Inclusivity
- 支援多種輸入方式（觸控、鍵盤、語音）
- 考慮不同能力的使用者需求
- 提供多語言支援

## 可及性標準 Accessibility

### WCAG 2.1 合規性

#### 色彩對比
- 一般文字：最小對比度 4.5:1 (AA)
- 大型文字（18px+）：最小對比度 3:1
- 重要文字：建議達到 7:1 (AAA)

#### 鍵盤導航
- 所有互動元素可通過鍵盤訪問
- 明確的焦點指示器
- 合理的 Tab 順序
- 提供跳過重複內容的連結

#### 螢幕閱讀器支援
- 使用語義化 HTML
- 提供適當的 ARIA 標籤
- 圖片包含替代文字
- 表單元素有明確的標籤關聯

#### 響應式設計
- 支援 320px 到 1920px 的視窗寬度
- 可縮放至 200% 不影響功能
- 觸控目標最小 44×44px
- 適當的點擊區域間距

### 測試檢查清單
- [ ] 色彩對比度符合標準
- [ ] 所有功能可通過鍵盤操作
- [ ] 螢幕閱讀器可正確朗讀內容
- [ ] 表單有清晰的錯誤提示
- [ ] 動畫可暫停或關閉
- [ ] 在不同裝置上測試通過
- [ ] 文字可放大至 200%
- [ ] 不依賴顏色作為唯一的信息傳遞方式

## 實施建議

### 開發團隊
1. 使用 CSS 變數管理設計令牌
2. 建立元件庫確保一致性
3. 實施自動化的可及性測試
4. 定期進行設計審查

### 設計團隊
1. 維護設計系統的 Figma 檔案
2. 定期更新設計指南文檔
3. 收集使用反饋並迭代改進
4. 進行可用性測試驗證設計決策

### 版本控制
- 設計系統採用語義化版本號（主版本.次版本.修訂版本）
- 重大變更提前通知並提供遷移指南
- 維護變更日誌記錄所有更新
